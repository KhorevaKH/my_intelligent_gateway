# Интеллектуальный шлюз для автоматизации входящего бумажного документооборота
**Все права защищены. Этот проект предоставлен только для просмотра и оценки. Никакая часть этого кода не может быть воспроизведена, изменена или распространена в любой форме без явного письменного разрешения автора**

Данный проект представляет собой прототип веб-приложения (интеллектуального шлюза), предназначенного для автоматизации обработки входящих бумажных документов в государственных органах. Система выполняет распознавание текста (OCR), семантический анализ (RAG + LLM), классификацию, маршрутизацию и автоматический контроль сроков исполнения.

## Ключевые особенности
- **OCR-распознавание:** Использование EasyOCR с GPU-ускорением для извлечения текста из сканов (JPG, PNG) и PDF.
- **Семантический анализ (RAG):** Векторный поиск по нормативной базе (Инструкция Минцифры) с помощью PostgreSQL + pgvector.
- **Классификация и маршрутизация:** Локальная LLM (Qwen2.5:3b через Ollama) определяет тип документа, срочность и назначает ответственный отдел.
- **Контроль сроков:** Автоматический расчет контрольных дат (3/15/30 дней) в соответствии с 59-ФЗ, цветовая индикация просрочек.
- **Безопасность:** Все компоненты развернуты локально (on-premise), реализовано журналирование действий пользователя.

## Технологический стек (Прототип)
- **Язык:** Python 3.11+
- **Веб-фреймворк:** Flask, Jinja2, Bootstrap 5
- **OCR:** EasyOCR (с поддержкой GPU)
- **LLM:** Qwen2.5:3b (Ollama)
- **Векторная БД:** PostgreSQL 16 + pgvector (расширение)
- **СУБД:** PostgreSQL 16 (единая база данных для метаданных)

## Инструкция по развертыванию (для локального запуска)

### 1. Предварительные требования
- Установленный Python 3.11+.
- Установленный Docker (для запуска PostgreSQL с pgvector).
- Установленный Ollama и загруженная модель Qwen2.5:3b (команда: `ollama pull qwen2.5:3b`).
- Установленный Git.

### 2. Клонирование и настройка окружения
```bash
git clone https://github.com/KhorevaKH/my_intelligent_gateway.git
cd my_intelligent_gateway
python -m venv venv
source venv/bin/activate  # для Linux/Mac
# или
venv\Scripts\activate  # для Windows
pip install -r requirements.txt
```

### 3. Запуск базы данных (PostgreSQL + pgvector)
Для работы RAG-модуля и хранения метаданных требуется единая БД. В корневой папке выполните команду для запуска Docker-контейнера:
```bash
docker run -d --name pgvector-container -e POSTGRES_USER=user -e POSTGRES_PASSWORD=pass -e POSTGRES_DB=intel_gateway -p 5433:5432 ankane/pgvector
```

### 4. Загрузка нормативной базы (для RAG)
Сначала убедитесь, что у вас есть файл `instruction.pdf` (Инструкция Минцифры) в папке проекта. Затем выполните скрипт загрузки:
```bash
python load_instruction_standalone.py
```

### 5. Запуск веб-приложения
```bash
python app.py
```
Приложение будет запущено по адресу http://127.0.0.1:5000
